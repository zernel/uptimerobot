## 1. 产品概述

UptimeRobot 是一个云端的网站和服务可用性监控平台，核心价值主张：

- **30秒内开始监控**：无需安装，注册即用
- **免费50个监控器**：永久免费层级
- **20+ 集成**：覆盖主流通知渠道
- **实时告警**：Web + 移动端推送
- **公开状态页面**：透明展示服务健康状态

**用户规模**：330万+ 用户，客户包括 Intel、EA、IBM、Stanford、BBC 等

## 2. 核心监控类型

### 2.1 HTTP(S) 监控
- 监控任何 HTTP/HTTPS 端点
- 检查 HTTP 状态码
- 支持自定义请求方法（GET、POST 等）
- 支持自定义请求头
- 可配置跟随重定向
- 支持基本认证

### 2.2 关键词监控
- 检查网页中是否存在/不存在特定关键词
- 支持两种条件：`关键词存在` 或 `关键词不存在`
- 条件触发时标记为"宕机"并发送通知
- 用途：检测页面内容损坏、错误信息、被篡改等

### 2.3 Ping 监控 (ICMP)
- 通过 ICMP Ping 检查服务器可达性
- 适用于服务器、网络设备、任何支持 Ping 的主机
- 用于基础网络连通性验证

### 2.4 端口监控 (TCP/UDP)
- 监控特定 TCP/UDP 端口的可用性
- 预设常见服务：SMTP(25)、DNS(53)、POP3(110)、FTP(21)、SSH(22)
- 支持自定义端口
- 可验证响应内容（关键词匹配）

### 2.5 API 监控
- 高级 API 端点监控
- 支持认证和自定义请求头
- 验证 JSON 响应中的关键字段和值
- 检测空结果、错误消息、异常响应
- 告警慢响应或断开的 API

### 2.6 UDP 监控
- 监控 UDP 服务（DNS、SNMP 等）
- 自定义目标端口
- 配置数据包响应规则
- 可选关键词验证响应

### 2.7 心跳/Cron 作业监控
- 反向监控模式：您的脚本向 UptimeRobot 发送信号
- 检查定时任务是否按时到达
- 适用于：备份、数据导入、报表生成、账单处理等
- 监控后台任务、无服务器任务、内部服务

### 2.8 SSL 证书监控
- 跟踪证书错误和过期风险
- 提前 30/14/7 天发送过期提醒
- 检测 HTTPS 访问问题

### 2.9 域名过期监控
- 跟踪域名到期风险
- 提前 30/14/7 天发送续费提醒
- 避免因忘记续费导致的宕机

### 2.10 DNS 监控
- 追踪特定 DNS 记录（A、AAAA、CNAME、MX）
- 检测记录变更或被删除
- 即时告警 DNS 篡改或配置错误
- 传统监控可能遗漏的额外保护层

### 2.11 响应时间监控
- 跟踪服务响应时间
- 设置 50ms - 5000ms 的阈值
- 响应变慢或恢复正常时告警
- 图表展示历史响应时间趋势

### 2.12 网站变更检测
- 追踪网页视觉变化
- 检测页面内容变动
- 适用于竞品监控、合规检查等

### 2.13 多位置监控
- 从多个全球位置验证可用性
- 4 个区域：北美、欧洲、亚洲、澳大利亚
- 设置区域特定的告警阈值
- 检测区域性故障、路由问题、CDN 边缘问题

## 3. 监控配置选项

### 3.1 监控间隔
| 计划 | 最小间隔 |
|------|----------|
| 免费 | 5 分钟 |
| Solo/Team | 60 秒 |
| Enterprise | 30 秒 |

### 3.2 请求配置
- 自定义 HTTP 方法
- 自定义请求头
- 请求超时设置（最长 30 秒）
- 跟随重定向选项
- 自定义成功状态码

### 3.3 告警触发规则
- 连续失败次数阈值（减少误报）
- 配置"告警延迟"（如宕机 X 分钟后才告警）
- 配置告警重复频率（每 Y 分钟重复告警）

### 3.4 维护窗口
- 预设维护时间段
- 暂停监控，避免误报
- 维护期间的数据不影响正常运行时间统计

## 附录 B：API v3 技术规格

### B.1 认证方式
- Bearer Token (JWT) 放在 `Authorization` 头部
- 3 种密钥类型：
  - **账户级**：所有 API 方法，所有监控器
  - **监控器级**：仅 `GET /monitors/{id}`
  - **只读**：所有 `GET*` 端点

### B.2 速率限制
| 计划 | 限制 |
|------|------|
| 免费 | 10 请求/分钟 |
| Pro | monitor_limit × 2 请求/分钟（最大 5,000） |

**响应头**：`X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset`、`Retry-After`

### B.3 核心端点

| 资源 | 端点 | 方法 |
|------|------|------|
| 监控器 | `/monitors`, `/monitors/{id}` | GET, POST, PATCH, DELETE |
| 监控器操作 | `/monitors/{id}/pause`, `/monitors/{id}/start`, `/monitors/{id}/reset` | POST |
| 批量操作 | `/monitors/bulk/pause`, `/monitors/bulk/start`, `/monitors/bulk/update` | POST |
| 监控器组 | `/monitor-groups`, `/monitor-groups/{id}` | GET, POST, PATCH, DELETE |
| 事件 | `/incidents`, `/incidents/{id}` | GET |
| 事件评论 | `/incidents/{id}/comments`, `/incidents/{id}/comments/{commentId}` | GET, POST, PATCH, DELETE |
| 公开状态页 | `/psps`, `/psps/{id}` | GET, POST, PATCH, DELETE |
| PSP 公告 | `/psps/{pspId}/announcements` | GET, POST |
| 维护窗口 | `/maintenance-windows`, `/maintenance-windows/{id}` | GET, POST, PATCH, DELETE |
| 告警联系人 | `/alert-contacts`, `/alert-contacts/{id}` | GET, POST, PATCH, DELETE |
| 标签 | `/tags`, `/tags/{id}` | GET, POST, PATCH, DELETE |
| 集成 | `/integrations`, `/integrations/{id}` | GET, POST, PATCH, DELETE |
| 用户 | `/user/me`, `/user/alert-contacts` | GET |
| 运行时间统计 | `/monitors/uptime-stats`, `/monitors/{id}/stats/uptime` | GET |
| 响应时间 | `/monitors/{id}/stats/response-time`, `/monitors/{id}/stats/response-time/all` | GET |

### B.4 分页方式
- 基于游标的分页（v3 新特性，替代 v2 的 offset/limit）

### B.5 响应格式
- 仅支持 JSON（v3 移除了 XML/JSON-P 支持）

### B.6 基础设施即代码
- **Terraform Provider**：官方发布，可管理监控器、集成、维护窗口、状态页
- **MCP 集成**：Model Context Protocol 支持（目前只读），支持 OAuth 或 API 密钥认证
